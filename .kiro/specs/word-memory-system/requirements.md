# 需求文档

## 介绍

单词记忆系统是一个基于艾宾浩斯遗忘曲线的单词学习和复习管理系统。该系统允许用户批量导入单词，并自动为每个单词生成科学的复习时间安排，帮助用户更有效地记忆和巩固词汇。

## 需求

### 需求 1

**用户故事：** 作为一个英语学习者，我希望能够通过上传txt文件批量导入每天学习的单词，这样我就可以高效地管理我的词汇学习。

#### 验收标准

1. 当用户在早上（如4月20日）上传包含单词数据的txt文件时，系统应当解析文件内容并将单词存储到数据库中，数据结构如下：

   **单词表 (words)**：
   - 英文单词（主键）
   - 中文释义（必填）
   - 音标（可选）
   - 词性（必填，从txt文件解析）
   - 例句（可选）
   - 导入时间（自动生成）
   
   **复习规则表 (review_rules)**：
   - 规则ID（主键）
   - 规则名称（A、B、C、D、E、F、G）
   - 时间偏移量（如：当天、1天后、2天后等）
   - 具体时间（如：14:00、21:00、7:00）
   - 是否启用（布尔值）
   - 创建时间
   
   **学习状态表 (word_study_records)**：
   - 记录ID（主键）
   - 英文单词（外键，关联单词表）
   - 复习时间段A（基于规则A计算的时间段，如：4月20日14:00-18:00）
   - 复习时间段B（基于规则B计算的时间段，如：4月20日21:00-23:59）
   - 复习时间段C（基于规则C计算的时间段，如：4月21日00:00-23:59）
   - 复习时间段D（基于规则D计算的时间段，如：4月22日00:00-23:59）
   - 复习时间段E（基于规则E计算的时间段，如：4月24日00:00-23:59）
   - 复习时间段F（基于规则F计算的时间段，如：4月27日00:00-23:59）
   - 复习时间段G（基于规则G计算的时间段，如：5月5日00:00-23:59）
   - 完成状态（JSON字段，记录A-G各时间段的打卡状态）
2. 当txt文件上传成功时，系统应当自动为每个导入的单词生成对应的学习状态记录，包含七个复习时间点，并生成JSON格式的遗忘曲线数据，格式如下：
   ```json
   {
     "04-20 14:00": "待打卡",
     "04-20 21:00": "待打卡", 
     "04-21 07:00": "待打卡",
     "04-22 07:00": "待打卡",
     "04-24 07:00": "待打卡",
     "04-27 07:00": "待打卡",
     "05-05 07:00": "待打卡"
   }
   ```
3. 当单词导入成功时，系统应当显示导入成功的确认信息和导入的单词数量
4. 如果txt文件格式不正确，系统应当显示具体的错误信息和格式要求
5. 当用户导入重复单词时，系统应当提供选择是否覆盖或跳过的选项

### 需求 2

**用户故事：** 作为一个英语学习者，我希望系统能够自动为每个单词生成基于科学记忆曲线的复习时间，这样我就可以在最佳时机复习单词以提高记忆效果。

#### 验收标准

1. 当单词被导入时，系统应当自动生成七个复习时间点：
   - A时间点：当天14:00
   - B时间点：当天21:00  
   - C时间点：1天后7:00
   - D时间点：2天后7:00
   - E时间点：4天后7:00
   - F时间点：7天后7:00
   - G时间点：15天后7:00
2. 当生成复习时间时，系统应当考虑导入时间，确保时间计算的准确性
3. 如果计算出的复习时间已经过去，系统应当将该复习时间设置为下一个可用的时间点
4. 系统应当将所有复习时间以标准时间格式存储在数据库中

### 需求 3

**用户故事：** 作为一个英语学习者，我希望能够查看我的单词和对应的复习时间安排，这样我就可以了解我的学习进度和计划。

#### 验收标准

1. 当用户请求查看单词列表时，系统应当显示所有已导入的单词
2. 当用户查看单词详情时，系统应当显示该单词的所有七个复习时间点
3. 当用户查看复习计划时，系统应当按时间顺序显示即将到来的复习任务
4. 系统应当区分显示已完成和未完成的复习任务

### 需求 4

**用户故事：** 作为一个英语学习者，我希望系统能够通过接口获取当前需要复习的单词，并确保我按照科学的记忆曲线进行复习，这样我就可以真正有效地记忆单词。

#### 验收标准

1. 当调用复习接口时，系统应当获取当前真实时间并与学习状态表中的复习时间进行比对
2. 当比对发现有到期的复习时间时，系统应当返回对应的单词列表供用户复习
3. 当用户完成复习时，系统应当记录复习完成状态和时间到JSON字段中
4. 当用户错过连续的复习时间点时，系统应当实施以下策略：
   - 如果用户错过1-2个连续时间点，允许在下一个时间点补充复习，但需要额外复习错过的内容
   - 如果用户错过3个或以上连续时间点，系统应当重置该单词的复习计划，从当前时间重新开始计算七个时间点
   - 系统应当在JSON字段中记录重置次数，用于分析学习效果
5. 当用户在规定时间窗口内（如时间点前后2小时内）完成复习时，系统应当标记为正常完成
6. 当用户超出时间窗口但在下一个时间点之前完成复习时，系统应当标记为延迟完成
7. 系统应当提供学习统计功能，显示用户的按时完成率和重置次数，帮助用户了解学习效果

### 需求 5

**用户故事：** 作为一个英语学习者，我希望能够自定义复习时间规则，这样我就可以根据自己的学习习惯调整复习计划。

#### 验收标准

1. 当用户访问规则管理页面时，系统应当显示所有复习规则（A-G）的当前配置
2. 当用户修改复习规则时，系统应当允许修改时间偏移量和具体时间
3. 当用户保存规则修改时，系统应当验证规则的有效性并更新数据库
4. 当规则被修改时，系统应当提示用户是否要应用到现有的学习记录

### 需求 6

**用户故事：** 作为一个英语学习者，我希望系统能够安全可靠地存储我的学习数据，这样我就不会丢失我的学习进度。

#### 验收标准

1. 当系统运行时，应当确保数据库连接的稳定性和可靠性
2. 当进行数据操作时，系统应当实施适当的错误处理和数据验证
3. 如果数据库操作失败，系统应当提供清晰的错误信息和恢复建议
4. 系统应当定期备份用户的学习数据以防止数据丢失
